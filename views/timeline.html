FALL 2025 — EENG 489

Project Proposal & Initial Research ✓ Done

Team formation, project scope definition, literature review on inductive sensing and gesture-controlled robotics. Initial component selection and budget planning.

Submitted capstone proposal — Smart Glove for pharmaceutical lab remote control
Identified LDC1614EVM as the inductance-to-digital converter
Defined 4 target gestures: Rest, Pinch, Flexion/Extension, Adduction
Selected Seeed Xiao ESP32-C6 for both glove and arm microcontrollers
FALL 2025 — EENG 489

Glove Fabrication & Sensor Design ✓ Done

Construction of the physical glove with sewn conductive thread coils. Sensor placement based on anatomical joint positions.

5 inductive coil sensors sewn: thumb, index, middle, ulnar, wrist crease
Coil design: 3–5 loops per sensor, ~2mm loop spacing for max inductance
Sensor placement spans from MCP to DIP joints for full range motion capture
Tested single-thread vs. stitched multi-segment coils for signal integrity
FALL 2025 / SPRING 2026

LDC1614 I2C Driver & PCB Integration ✓ Done

Wrote the low-level I2C driver for the LDC1614EVM. Configured all 4 channels and debugged the CONFIG register issue that caused all-zero outputs.

Identified critical CONFIG bug: 0x16 vs 0x14 — solved all-zero channel issue
All 4 channels reading in 59–63 million counts range at rest
I2C address confirmed at 0x2A — ADDR pin tied internally to GND on EVM
Wiring standard: D4/GPIO21 = SDA (blue), D5/GPIO23 = SCL (green)
SPRING 2026 — Week 7

Data Collection & Calibration ✓ Done

Systematic collection of REST and PINCH gesture data using sound-cued sessions. Inductor configuration optimized for clean signals.

Tested single 470µH inductor per finger → switched to two in series for cleaner data
REST baselines: Thumb ~4.231MHz, Index ~1.470MHz, Middle ~28.79MHz
10 sessions × 10 seconds each for both REST and PINCH gestures
Normalized pinch values: Thumb −0.507, Index −2.060, Middle −2.446
SPRING 2026

ML Pipeline & Neural Network Training ✓ Done

Built complete Python training pipeline. Trained 9→16→8→2 feedforward network. Exported weights as C++ header for on-device inference.

Feature extraction: mean, std deviation, slope per channel over 15-sample window
Network: 314 parameters, ~1.3KB Flash, <1ms inference at 160MHz
Achieved 100% training accuracy on collected gesture data
Weights exported to gesture_model_weights.h for GloveDemoML.ino
SPRING 2026

ESP-NOW Wireless Link & Motor-Based Arm Receiver ✓ Done

Replaced servo-based arm control with DC motors driven by an H-bridge. Motors provide significantly more torque than hobby servos, making them better suited to physically moving the robotic finger mechanisms under load. The arm receiver firmware was rewritten to control motor speed via PWM and direction via H-bridge logic pins.

ESP-NOW protocol retained: 11-byte packet, <5ms latency, gesture + intensity + raw channel values
Switched from PCA9685 servo driver to H-bridge motor driver — better torque for finger actuation
DC motors chosen over servos: higher torque output, simpler mechanical coupling to finger tendons
PWM duty cycle maps pinch intensity 0–100% directly to motor speed 0–255
H-bridge direction pins set FORWARD on PINCH, REVERSE on REST to open fingers back to position
Deadband control prevents motor jitter when pinch intensity is held steady at a grip
SPRING 2026 — MARCH 2026

Dual-Phase Calibration Firmware (GloveDemoV3) ✓ Done

Upgraded the glove firmware from a fixed-threshold system to a two-phase personal calibration system. The old approach used hardcoded count values that had to be manually re-tuned for each user and glove position. The new system adapts automatically to the user's actual hand range at every startup — no code changes required between users.

Phase 1 — Rest: 100 samples over 10 seconds with hand open and flat. Captures resting inductance baseline per channel.
Phase 2 — Pinch: 100 samples over 10 seconds at full hard pinch. Records peak delta per channel as the personal grip range.
Detection threshold automatically set to 25% of measured peak delta — adapts to every user and glove fit
Intensity now scaled to personal peak: 0% = rest baseline, 100% = your full squeeze. No hardcoded numbers.
Serial Monitor prints both calibration summaries so results can be verified before the demo begins
Calibration failure detection: if fewer than 10 valid samples are captured, firmware halts and reports the wiring issue
SPRING 2026 — FINAL

Final Prototype & Presentation ⏳ In Progress

Full physical robotic arm assembly with 3 DC motors controlled by the H-bridge receiver. End-to-end live demonstration with both calibration phases, wireless gesture transmission, and proportional motor-driven finger movement. Final written report and oral presentation to faculty and judges.

3× DC motors — one per finger, selected for torque advantage over hobby servos in finger tendon actuation
H-bridge motor driver controls speed via PWM and direction via logic pins from the Arm Xiao
Full demo flow: dual-phase calibration → live gesture detection → ESP-NOW transmission → motor response
Final report to include ML validation accuracy, system architecture, and calibration methodology
Oral presentation to NYIT faculty and senior design judges — Spring 2026
