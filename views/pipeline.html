System Pipeline

Data Flow:
Bend to Move

Every step from a finger bending to a DC motor responding — hardware, firmware, wireless protocol, and gesture logic explained.

Full Signal Chain
🧤

Glove Coil

Conductive thread coil changes magnetic inductance as the finger bends. Inductance shift is proportional to the degree of bend.

📊

LDC1614EVM

Converts LC resonant frequency to a 28-bit integer at 10Hz. Four channels scanned in sequence. I2C at 0x2A. Error values (4294967295, 268435455) are filtered out using the last good value.

🔵

Glove Xiao — Classification

Auto-calibrates at startup (60 samples / 6 seconds). Computes delta from rest baseline per channel. 2-of-3 bent fingers = PINCH. Intensity = average delta as % of threshold sum. Outputs MATLAB CSV + human-readable status line.

📶

ESP-NOW Wireless

11-byte packet sent to arm Xiao MAC address. Contains gesture (REST/PINCH), intensity (0–100%), and raw CH0/CH1/CH2 values. Latency under 5ms. Both boards must be on the same WiFi channel.

🔵

Arm Xiao — Motor Control

Maps pinch intensity (0–100%) to a PWM duty cycle sent to the H-bridge. Direction pins set to CLOSE on PINCH, OPEN on REST. LED blinks on each received packet. Prints machine-readable line for Python dashboard.

⚙️

H-Bridge + DC Motors

H-bridge receives PWM speed signal and direction bits from the Arm Xiao. Three DC motors drive the robotic fingers proportionally — higher pinch intensity = faster motor = tighter grip. Reversing direction pins opens the fingers back to rest position.

⚙️

Servos

Three SG90/MG90S servos move proportionally to pinch intensity. In the new build, the PCA9685 I2C driver controls them via angleToPulse() mapping (0–180° → SERVOMIN–SERVOMAX).

Gesture Detection Math

Step 1 — Calibration

60 samples taken at startup with hand open and flat. Average = rest baseline per channel.

rest_0 = avg(CH0 samples during 6s) rest_1 = avg(CH1 samples during 6s) rest_2 = avg(CH2 samples during 6s) // If good samples < 10 → calibration fails

Step 2 — Delta Calculation

Every loop, compute deviation from baseline.

d0 = CH0 − rest_0 // Thumb delta d1 = CH1 − rest_1 // Index delta d2 = CH2 − rest_2 // Middle delta

Step 3 — PINCH Classification

2-of-3 rule: any two channels above threshold = PINCH.

if abs(d0) > 2,000,000 → bent++ if abs(d1) > 1,000,000 → bent++ if abs(d2) > 2,000,000 → bent++ if bent >= 2 → PINCH DETECTED

Step 4 — Intensity → Motor Speed

Continuous proportional control — pinch harder, motors spin faster, grip tightens.

avgDelta = (|d0| + |d1| + |d2|) / 3 intensity = avgDelta × 100 / sum_of_thresholds // Map to PWM duty cycle: pwm_speed = map(intensity, 0, 100, 0, 255) // 0% pinch = 0 PWM → motor stopped // 100% pinch = 255 PWM → full speed close // Direction control: // PINCH → IN1=HIGH, IN2=LOW (close) // REST → IN1=LOW, IN2=HIGH (open)
ESP-NOW Packet Structure

GesturePacket — 11 bytes

typedef struct GesturePacket { uint8_t gesture; // 0=REST, 1=PINCH uint8_t intensity; // 0..100 (%) uint32_t ch0; // Raw thumb reading uint32_t ch1; // Raw index reading uint32_t ch2; // Raw middle reading } GesturePacket; // Total: 11 bytes
