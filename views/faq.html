Hardware & Sensors
This is almost always a CONFIG register issue. Find the line writeConfig(LDC_ADDR, 0x1A, 0x16, 0x01) and change 0x16 to 0x14. The value 0x16 enables external-clock mode — the chip waits for an external oscillator that isn't present and outputs zeros indefinitely. After changing, reflash the firmware. If channels still read 4294967295, a specific coil is open-circuit — check the solder joint at the sensor wire ends and confirm the coil wires are seated in both INxA and INxB.
That specific coil's wires are not seated properly in the EVM terminals. Push them firmly into INxA and INxB on the EVM board. Also check that both wire ends of the coil are connected — each coil needs two connections, one to INxA and one to INxB. If wires look fine, inspect the coil itself for a break in the conductive thread.
The coil may not be close enough to a metal surface inside the glove, or the coil geometry needs adjustment. Ensure there is a metal target (foil or metal object) inside the finger and the coil can sense it. Also verify the coil has at least 3–5 loops with ~2mm spacing — fewer loops or wider spacing weakens the signal dramatically. Two 470µH inductors in series per finger significantly improves signal quality.
The LDC1614EVM evaluation board has the ADDR pin already tied to GND internally via an onboard resistor on the PCB. This sets the I2C address to 0x2A automatically. You do not need to run an external wire from GND to ADDR — it is already done for you. If you add an external wire, you risk creating a ground loop. Trust the EVM board's design.
Three servos moving simultaneously can draw 500mA or more, which brownouts the Xiao's onboard 5V regulator and causes a reset. Never power all three servos from the Xiao's 5V pin. Use an external 5V power supply (a phone charger with a breakout, a bench supply, or a USB power bank). Connect the GND of the external supply to the same GND rail as the Xiao — a shared common ground is essential.

PINCH_THRESHOLD is set too high for your glove. Open Serial Monitor and note the channel values at rest. Then pinch fully and note the new values. The difference between those two numbers is your signal range. Set PINCH_THRESHOLD for each channel to about 30–40% of that delta. For the index finger (CH1), the threshold is already lower at 1,000,000 — try 500,000 if still not detecting. Also make sure calibration ran correctly — the baselines may be off if the hand wasn't flat during the 6-second countdown.
Two possibilities: (1) PINCH_THRESHOLD is too low — raise it. Try doubling the current value. (2) Calibration ran while the hand was not fully open, so the baselines are captured incorrectly. Reflash the firmware — the auto-calibration runs fresh on every upload. During the 6-second calibration countdown, hold the hand as flat and relaxed as possible on a table surface.

Arduino & Firmware
You likely installed "Seeed nRF52 Boards" instead of "Seeed ESP32 Boards." In Boards Manager, uninstall any nRF52 package. Make sure the correct URL is in Settings → Additional boards manager URLs: https://files.seeedstudio.com/arduino/package_seeeduino_boards_index.json. Then search "Seeed ESP32" and install "Seeed ESP32 Boards" by Seeed Studio. Restart Arduino IDE and try again.
ArmReceiver.ino is in the same folder as GloveDemo.ino. Arduino compiles all .ino files in a folder together. Move ArmReceiver.ino into its own subfolder: GloveDemo/ArmReceiver/ArmReceiver.ino. Open it as a separate sketch by navigating into that subfolder in Arduino IDE. Never open it from the GloveDemo root folder.
Install the ESP32Servo library: Arduino IDE → Tools → Library Manager → search "ESP32Servo" → install the entry by Kevin Harrington & John K. Bennett. Do NOT install the generic "Servo" library — it does not support the ESP32-C6. Only the ESP32Servo library (by Kevin Harrington) works correctly with this board.
On macOS, use pip3 instead of pip for all Python package installations. For Arduino IDE Settings on macOS, click "Arduino IDE" in the top-left menu bar (next to the Apple logo) and choose Settings — it is NOT under the File menu like on Windows.
